┌─ 🎯 KEY FINDINGS ───────────────────────────────────────────────────────────
│
│ CRITICAL BUG:
│ ─────────────
│ The POST /orders endpoint had a severe bug where product stock was reduced
│ in memory but NOT persisted to the database because the product object was
│ never added to the SQLAlchemy session before commit.
│
│ Example:
│ ├─ Order 1: product.stock 100 → 95 (in memory) → commit → DB still 100
│ ├─ Order 2: product.stock 100 → 95 (in memory) → commit → DB still 100  
│ └─ Result: Sold 10 units when only 5 existed = INVENTORY LOSS
│
│ Solution: One line added → db.add(product) before db.commit()
│
